Sticky Hero Demo

A Hero That Sticks, Content That Scrolls Over

This hero remains pinned to the top of the viewport as you scroll, but because it still participates in normal layout flow, the page content starts beneath it and then scrolls up over the hero. You can remove this description text and the hero height will shrink accordingly without needing any JS.

Section 1: Intro

Scroll down and watch how this content passes over the hero. The hero itself is implemented with position: sticky on its inner wrapper and a higher z-index on the content.

Inspect the DOM to see that the hero is still above this section in the normal document flow, so we don’t need any hard-coded margin-top or padding-top based on a fixed hero height.

Section 2: More Content

In your real project, if your sticky behavior isn’t working, check for:

  • An ancestor with overflow: hidden/auto that can clip sticky.
  • Missing top value on the sticky element.
  • Not enough scrollable height in the sticky element’s container.

Section 3: Final Section

Delete or shorten the hero description in this demo and refresh to confirm that the hero height changes and the content still lines up correctly. That’s the core advantage of keeping the hero in the normal layout and using position: sticky instead of fully position: fixed.
